# 系统权限说明

> 本文档标记当前系统中各角色可执行的操作。系统采用三级角色层级：会长 > 超管 > 管理，上级自动继承下级所有权限。

---

## 1. 角色层级

```
会长（HUIZHANG, level 3）
  └─ 超管（CHAOGUAN, level 2）
       └─ 管理（GUANLI, level 1）
```

**继承规则**：用户角色等级 ≥ 接口所需最低角色等级即放行。例如 `requireRole(Role.CHAOGUAN)` 允许超管和会长访问。

---

## 2. 数据范围限制

除角色层级外，还有基于 `branchId` 的数据范围限制：

| 角色 | 数据可见范围 | 说明 |
|------|-------------|------|
| 会长 | 所有分部 | 可选指定厅，也可查看全部厅汇总 |
| 超管 | 仅本厅 | `branchId` 锁定到 `user.branchId`，无法查看其他厅 |
| 管理 | 仅本厅 | `branchId` 锁定到 `user.branchId`，无法查看其他厅 |

**关键限制**：
- 超管/管理无法通过 `viewAll` 参数查看全部厅汇总数据
- 超管访问历史记录仅限本厅
- 超管只能更新本厅奖励规则
- 超管厅列表仅显示本厅
- 会长删除厅时需输入登录密码确认
- 会长可操作其他会长账户（仍不能操作自己）
- 会长可设置任意角色（含会长），超管只能设置管理
- 会长添加会长时 `branchId` 可为 null

## 4. 业务规则汇总

### 4.1 备注系统

| 操作类型 | 备注要求 | 存储方式 |
|----------|----------|----------|
| 数据录入（单条/批量/导入） | **必填** | 覆盖式写入 `DataRecord.remark`，同时独立记录到 `DataHistory.remark` |
| 数据修改 | **必填** | 覆盖式写入 `DataRecord.remark`，同时独立记录到 `DataHistory.remark` |
| 数据删除 | **必填** | 仅记录到 `DataHistory.remark`（记录已删除） |
| 修改自己的昵称 | 不需要 | - |

**备注规范化**：
- `trim` 处理
- 限 100 字
- 空字符串归 `null`

**编辑无变化处理**：
- 编辑时若 sg/mx/qm/人员/冠名均无变化，后端返回 400「数据无变化，无需保存」
- 即使添加了新备注，也不会保存（数据必须实际变化才能保存）

### 4.2 历史记录

- 每次录入创建独立的 `DataHistory(CREATE)` 记录，不覆盖
- `DataRecord` 同一人同一周只保留一条（累加汇总）
- `DataHistory` 每次操作独立保留，记录增量值
- 历史记录页采用"卡片入口 + 详情视图"模式，三个卡片分别对应录入、修改、删除
- 历史记录页表格包含"备注"列

### 4.3 数据录入时区处理

- 数据录入接口接受可选 `weekStart` 参数（YYYY-MM-DD 格式，必须是周一）
- 周统计厅使用用户查看的周作为 `weekStart`，支持编辑历史周数据
- 月统计厅当前月录入到当前周，历史月录入到该月最后一周
- 数据录入时使用客户端时区计算当前周，避免服务器 UTC 时区导致跨凌晨数据归属错误

### 4.4 冠名转换

- 仅按月统计的厅启用冠名转换
- 录入收光时按阈值整除转换为冠名，余数计入收光
- 多等级时按 `threshold` 降序逐级扣减转换
- 冠名数量累加存储在 `DataRecordNaming` 关联表

### 4.5 福利扣减

- 独立扣减表（`Deduction`），与 `DataRecord` 解耦
- 按周期记录每个人员的福利扣减金额
- 最终福利 = 原福利 - 扣减
- 周统计厅：`periodStart` = 该周周一
- 月统计厅：`periodStart` = 该月 1 号

## 7. 权限校验实现

### 7.1 中间件

权限校验通过两个中间件实现：

```typescript
// 认证中间件：验证 JWT
{ preHandler: [authenticate] }

// 角色校验中间件：验证角色等级
{ preHandler: [authenticate, requireRole(Role.GUANLI)] }        // 管理+
{ preHandler: [authenticate, requireRole(Role.CHAOGUAN)] }      // 超管+
{ preHandler: [authenticate, requireRole(Role.HUIZHANG)] }       // 仅会长
{ preHandler: [authenticate, requireRole(Role.HUIZHANG, Role.CHAOGUAN)] }  // 会长或超管
```

### 7.2 数据范围校验

业务逻辑层通过 `resolveBranchId` 函数校验数据范围：

```typescript
// 会长：可指定任意 branchId，未指定时返回 null
// 非会长：只能操作自己 branchId，指定其他 branchId 返回 403
```

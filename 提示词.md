# 全栈统计系统 — 需求规格说明

## 一、项目概述

开发一个面向组织内部的**数据统计与福利计算系统**，支持多分部管理、三级权限控制、周度数据统计、自动福利计算与排名、数据可视化看板等功能。系统面向中型用户规模（10-100人），支持本地部署与服务器部署。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| 分部 | 组织内的下属单位，由会长创建和管理 |
| 人员 | 由超管添加的成员名单（仅姓名，非系统账户），一人可属于多个分部 |
| 账户 | 系统登录凭证，分为会长、超管、管理三个角色 |
| 收光 | 人员数据字段，整数，表示收到的光芒数量 |
| 麦序 | 人员数据字段，整数，表示麦克风排序次数，用于周排名 |
| 全麦 | 人员数据字段，整数，表示全场麦克风次数 |
| 福利 | 根据收光、全麦及排名奖励自动计算的积分 |
| 统计周期 | 每周一至周日为一周，各周数据独立统计，不累计 |

---

## 三、权限管理

### 3.1 角色与权限矩阵

| 功能 | 会长 | 超管 | 管理 |
|------|:----:|:----:|:----:|
| 添加/禁用/删除超管账户 | ✓ | - | - |
| 添加/禁用/删除管理账户 | ✓ | ✓（本分部） | - |
| 增删分部 | ✓ | - | - |
| 管理本分部人员名单 | ✓ | ✓ | - |
| 录入本分部人员数据 | ✓ | ✓ | ✓ |
| 修改已录入数据 | ✓ | ✓ | ✓ |
| 删除已录入数据 | ✓ | ✓ | - |
| 自定义奖励规则 | ✓ | ✓ | - |
| 查看本分部数据 | ✓ | ✓ | ✓ |
| 数据导出 | ✓ | ✓ | ✓ |
| 查看数据看板 | ✓ | ✓ | ✓ |

> 会长拥有系统所有权限。权限继承：上级自动拥有下级的所有权限。

### 3.2 账户规则

- 登录方式：用户名 + 密码
- 密码无复杂度要求，首次登录不强制修改密码
- 超管仅可管理自己分部内的管理账户
- 界面语言：仅中文

### 3.3 分部规则

- 分部下存在数据时禁止删除，需先转移或清理关联数据
- 各角色仅可查看自己所属分部的数据

---

## 四、数据管理

### 4.1 人员名单

- 由超管在分部内添加，仅需填写姓名
- 人员名单与系统账户无关
- 同一人可同时属于多个分部

### 4.2 数据录入

**录入字段**（均为非负整数）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 收光 | 整数 | 收到的光芒数量 |
| 麦序 | 整数 | 麦克风排序次数 |
| 全麦 | 整数 | 全场麦克风次数 |

**录入方式**：

| 方式 | 说明 |
|------|------|
| 手动输入 | 表单录入，支持单人和批量录入 |
| Excel导入 | 上传Excel文件批量导入 |
| 表格粘贴 | 直接粘贴表格数据批量导入 |

**录入规则**：

- 随时录入，系统自动按周（周一至周日）归类
- 每周数据独立统计，不累计
- 录入时进行数据验证，确保字段完整且为非负整数

### 4.3 数据修改与删除

- **修改**：录入后可修改，系统自动记录修改历史（修改人、修改时间、修改前值、修改后值）
- **删除**：仅会长和超管可删除，删除操作同样记录在修改历史中

### 4.4 历史数据

- 支持查看历史各周数据
- 支持不同周的数据对比

---

## 五、福利计算与排名

### 5.1 福利计算规则

以下为默认规则，超管可在分部内自定义调整：

| 计算项 | 默认规则 | 说明 |
|--------|----------|------|
| 收光福利 | 1收光 = 3福利 | 可由超管自定义转换比例 |
| 全麦福利 | 1全麦 = 3福利 | 可由超管自定义转换比例 |

**个人总福利 = 收光福利 + 全麦福利 + 排名奖励福利**

### 5.2 排名机制

- 排名范围：分部内，按麦序数值降序排列
- 排名周期：每周独立排名

### 5.3 排名奖励

以下为默认奖励值，超管可在分部内自定义调整：

| 奖励项 | 默认值 | 说明 |
|--------|--------|------|
| 周排名第1名 | 超管自定义 | 额外福利点数 |
| 周排名第2名 | 超管自定义 | 额外福利点数 |
| 周排名第3名 | 超管自定义 | 额外福利点数 |
| 麦序≥40但未进前3名 | 52福利 | 达标奖励，超管可自定义 |

### 5.4 自动计算

- 系统根据录入数据自动计算每人福利
- 奖励规则变更后，已有数据自动重新计算

## 六、数据看板

- 所有角色可见（仅限本分部数据）
- 包含以下模块：

| 模块 | 内容 |
|------|------|
| 本周汇总 | 分部内本周收光/麦序/全麦的汇总统计图表 |
| 排名展示 | 分部内周排名前3名及麦序达标人员展示 |
| 周对比 | 本周与上周的数据对比图表 |

---

## 七、数据导出

- 导出格式：Excel、CSV
- 导出内容：周报数据（含人员数据与福利统计）
- 导出权限：所有角色可导出本分部数据

---

## 八、系统通知

| 通知场景 | 触发条件 | 通知对象 |
|----------|----------|----------|
| 周排名公布 | 每周统计周期结束 | 本分部所有账户 |
| 奖励规则变更 | 超管修改奖励规则 | 本分部所有账户 |
| 数据变更 | 数据被修改或删除 | 本分部相关账户 |

---

## 九、非功能需求

| 类别 | 要求 |
|------|------|
| 用户界面 | 直观友好，响应式设计，支持不同设备访问 |
| 数据安全 | 敏感数据加密存储，完善的身份验证机制 |
| 数据备份 | 定期自动备份，防止数据丢失 |
| 性能 | 数据计算高效，页面加载迅速 |
| 可扩展性 | 系统架构支持未来功能扩展 |
| 部署 | 支持本地部署和服务器部署 |

## 十、技术栈

- 前端：React 或 Vue
- 后端：Node.js 或 Python
- 数据库：MySQL 或 PostgreSQL
- 本地开发可使用本地数据库
